# YAML 格式

YAML是一种人类可读的数据序列化格式，主要用于配置文件和数据交换，其语法使用缩进表示层级关系，并支持多种数据结构。

## 1. 核心特点

YAML的核心特点：

1. 以数据为中心，强调可读性：YAML设计初衷是便于人类阅读和编写，使用简洁的语法（如缩进、避免过多封闭符号）来表达数据结构，相较于XML和JSON更直观。
2. 与JSON高度兼容：JSON的语法是YAML 1.2版的子集，因此大多数JSON文件都可以被YAML解析器读取。但YAML支持更丰富的特性，如锚点引用、字符串无需引号等。
3. 广泛的应用支持：YAML被多种编程语言和现代工具支持，常见于Spring Boot、Docker、Kubernetes等系统的配置文件，文件后缀通常为`.yml`或`.yaml`。

## 2. 基本语法与结构

YAML主要使用以下三种结构来表示数据：

1. 标量 (Scalars)：最基本的不可再分值，如字符串、布尔值、整数、浮点数、Null、日期等。
    - 字符串可以不用引号，但需注意单引号不支持转义字符，双引号支持。
    - 可以使用`!!`（如`!!str`）显式指定数据类型。
2. 序列 (Sequence)：即数组或列表，使用短横杠`-`表示列表项，且项前需缩进。
3. 映射 (Mapping)：即键值对或对象，使用冒号`:`分隔键和值，冒号后必须有一个空格。

书写时需遵循以下关键规则：

1. 使用空格缩进表示层级关系，同一层级元素必须左对齐。不能使用Tab键进行缩进。
2. 文件可由多个文档组成，文档间用`---`分隔，文档结束可选`...`。
3. 支持注释，以``开头。
4. 支持锚点（`&`）和引用（`*`）以实现数据复用。
